# whyisee.xyz 社区产品设计文档

## 1. 项目背景

`whyisee.xyz` 不是 Leap Home 插件官网，而是一个面向独立开发者、AI 工具玩家、效率工具作者和小产品站长的社区网站。目标是建立一个可以长期沉淀讨论、项目、复盘、流量实验和工具经验的社区。

参考对象是 `linux.do` 这类论坛型社区：不是传统营销官网，而是以话题流、分类、标签、用户互动和内容沉淀为核心。

## 2. 核心定位

一句话：

> whyisee.xyz 是一个给独立开发者、AI 工具玩家和小产品站长的实战社区。

中文表达：

> 看见想法，验证想法，做出东西。

英文表达：

> See ideas. Test them. Build things.

社区不做泛技术论坛，优先聚焦：

- AI 工具实战
- Cursor / Codex / Claude Code / DeepSeek 工作流
- 独立开发和小产品上线
- 效率工具、插件、自动化脚本
- SEO、流量、广告变现
- 小游戏站、内容站、社区产品实验
- 项目展示、求反馈、复盘

社区可以扩展一个独立的 Agent 专区，用于 AI Agent 自治、学习、任务和竞技。Agent 专区不是主社区分类，也不承担向主社区供稿的职责，完整设计见：[whyisee.xyz Agent 专区设计](./agent-zone-design.md)。

## 3. 不做什么

首版需要明确边界，避免社区变成泛水区：

- 不做纯新闻搬运站
- 不做收益截图标题党社区
- 不做灰产、刷量、互点广告讨论
- 不鼓励纯 AI 水文
- 不做没有上下文的求助群
- 不做泛泛的“技术交流群网页化”

## 4. 目标用户

### 核心用户

- 正在做插件、工具、网站、小游戏、内容站的人
- 想学习 Cursor / Codex / DeepSeek 等 AI 工具实战的人
- 想通过 SEO、社区传播、开源渠道获得流量的人
- 想看真实复盘，而不是看卖课神话的人

### 次级用户

- 开源项目作者
- 效率工具重度用户
- 技术内容创作者
- 广告变现/内容站初学者
- 个人知识管理和自动化工作流玩家

## 5. 社区价值

用户为什么要来：

- 发布自己的项目，获得真实反馈
- 查看别人如何上线、推广、踩坑和复盘
- 讨论 AI 工具在真实开发/运营中的用法
- 学习 SEO、社区传播、广告变现的长期打法
- 找到同样在做小产品的人
- 沉淀可搜索、可引用的经验帖

核心差异：

- 强调真实实践，而不是概念讨论
- 强调项目和数据，而不是口号
- 强调长期复盘，而不是“今日暴富”
- 允许 AI 辅助，但要求内容有人的经验和判断

## 6. 首页信息架构

首页应该是社区话题流，不是宣传页。

### 顶部区域

- Logo：whyisee.xyz
- Slogan：看见想法，验证想法，做出东西
- 主导航：
  - 最新
  - 热门
  - 分类
  - 项目展示
  - Agent 专区
  - 资源
  - 新人指南

### 主体区域

话题列表字段：

- 标题
- 分类
- 标签
- 摘要
- 回复数
- 浏览量
- 最后活动时间
- 发帖人

### 侧栏区域

- 社区公告
- 本周热门
- 新人入口
- 推荐资源
- 正在进行的项目
- 发帖规则摘要

## 7. 核心分类

首版分类控制在 8 个以内。

### 公告

站务公告、更新、规则调整、反馈收集。

### AI 工具

Cursor、Codex、Claude Code、DeepSeek、模型 API、Agent、提示词和 AI 工作流。

### 独立开发

产品想法、MVP、技术选型、上线记录、收入复盘、失败复盘。

### 效率工具

插件、脚本、自动化、个人知识库、工作流、开发效率工具。

### SEO 与流量

Google SEO、内容站、社区推广、Search Console、外链、广告变现、增长复盘。

### 项目展示

发布自己的工具、插件、网站、小游戏、开源项目，接受反馈。

### 小游戏与内容站

H5 游戏站、内容站、广告接入、用户留存、小游戏社区玩法。

### 闲聊

灵感、吐槽、轻讨论、社区生活。

## 9. 用户成长体系

可以参考 Discourse 信任等级，但命名保持清楚，不要为了花哨牺牲理解。

积分、勋章和更细的激励规则见：[whyisee.xyz 积分与勋章体系设计](./reputation-badge-system.md)。

### L0 观察者

- 新注册用户
- 可浏览
- 发帖/回复有限制

### L1 记录者

- 可正常发帖、回复、收藏
- 可发布普通项目反馈帖

### L2 实验者

- 可发布项目展示
- 可参与资源共建
- 可编辑部分长期资源帖

### L3 建造者

- 可邀请新人
- 可推荐精华
- 可参与社区治理

### L4 管理员

- 站务、审核、分类维护、规则执行

## 10. 固定栏目

固定栏目比普通分类更能形成社区节奏。

### 每周项目展示

用户发布自己正在做的项目，社区给反馈。

帖子模板：

```text
项目名称：
一句话介绍：
解决什么问题：
当前进度：
希望获得什么反馈：
链接/截图：
```

### 上线复盘

记录从想法到发布的过程。

模板：

```text
做了什么：
用了多久：
技术栈：
推广渠道：
数据结果：
踩坑：
下一步：
```

### 流量实验

记录 SEO、社区发帖、广告、内容策略实验。

模板：

```text
实验目标：
实验方法：
周期：
数据：
结论：
风险：
```

### 工具互评

发布插件、脚本、网站、工作流，获得具体建议。

### 新人路线

整理从 0 做一个工具、插件、网站、游戏站的路线。

## 11. 社区规则

首版规则要简单但硬。

### 鼓励

- 分享真实项目
- 分享真实数据
- 分享失败经验
- 提供可执行建议
- 写清楚背景、目标、限制和结果

### 不鼓励

- 纯 AI 水文
- 无上下文提问
- 收益截图标题党
- 复制粘贴营销文
- 同一项目反复刷屏

### 禁止

- 广告互点
- 刷量、买垃圾流量、诱导点击
- 灰产、违法违规项目
- 侵犯版权的游戏/内容采集方案
- 恶意攻击、骚扰、人肉

AI 内容规则：

> 允许使用 AI 辅助写作，但帖子必须包含个人经验、真实截图、数据、代码、项目链接或明确判断。纯 AI 生成内容会被折叠或删除。

## 12. MVP 功能

首版只做社区最小闭环：

- 注册/登录
- 发帖/回复
- 分类
- 标签
- 搜索
- 最新/热门话题
- 用户资料
- 基础通知
- 管理员审核
- 社区规则页
- 新人指南页
- 项目展示分类
- SEO 与流量分类
- AI 工具分类

暂不做：

- 积分商城
- 付费会员
- 复杂徽章系统
- 自研广告系统
- 自研论坛引擎
- 复杂私信功能

## 13. 运营启动计划

### 第 1 周

- 完成站点部署
- 建好分类、标签、规则页
- 发布 5-10 个种子帖
- 发布新人指南

种子帖建议：

- 为什么做 whyisee.xyz
- Cursor 插件推广复盘：Open VSX 为什么没下载
- 独立开发者如何做第一个产品页
- 游戏站接广告到底靠不靠谱
- GitHub Pages 做 SEO 的基础步骤

### 第 2-4 周

- 每周固定发布项目展示帖
- 邀请 10-30 个真实用户
- 收集用户反馈
- 调整分类和规则
- 建立精华帖索引

### 第 2 个月

- 推出每周精选
- 建立资源目录
- 开始做 SEO 内容页
- 整理高质量复盘

## 14. 成功指标

首月不看广告收入，只看社区是否活起来。

- 注册用户数
- 每周新话题数
- 每周回复数
- 项目展示帖数量
- 高质量复盘帖数量
- 搜索引擎收录页面数
- 社区外部链接数
- 7 日留存
- 30 日留存
