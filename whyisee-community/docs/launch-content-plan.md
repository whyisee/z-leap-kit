# whyisee.xyz 上线前初始内容规划

## 1. 文档目标

本文档用于规划 `whyisee.xyz` 正式上线前的初始内容池。

初始内容不是为了把网站填满，而是给第一次访问的用户建立判断：

- 这个社区主要聊什么
- 内容质量是否值得继续看
- 用户能不能自然参与回复
- 这里是不是有人在认真维护

早期内容应该像一个有样板间的小社区，而不是一个批量生成的内容站。

正式内容重写必须先参考最新调研：[whyisee.xyz 当前环境与用户需求研究](./current-demand-research-2026-06.md)。后续所有种子内容以真实用户需求、近期工具变化和中文开发者现实约束为依据，不再按分类数量硬填内容。

## 2. 核心原则

### 2.1 内容是产品的一部分

社区上线时不能只上线功能。首页、分类页和话题详情页本身就是用户判断产品气质的入口。

首批内容要让用户感到：

- 话题聚焦在独立开发、AI 工具、效率工具、SEO 和小产品实践
- 帖子有具体问题、具体经验或明确取舍
- 讨论可以接下去，而不是读完就结束
- 站长在控制质量，而不是放任 AI 水文堆积

### 2.2 少而精优先

上线前不需要准备上百篇内容。太多初始内容如果质量不稳定，会让社区显得像空壳内容库。

当前第一版脚本曾准备 14 个模拟普通用户、50 篇左右内容和一批轻量互动数据，但这只能作为功能验证数据，不能直接作为正式上线内容池。

正式上线内容应缩到 25 篇左右：

- 14 个不可登录的模拟普通用户，用于分散首批内容作者和回复作者
- 8 篇左右卡住排查型内容，解决错误、部署、工具失控和流量卡点
- 5 篇左右工具选择型内容，帮助用户判断 Cursor、Codex、Claude Code、DeepSeek、Trae 等如何分工
- 5 篇左右方向判断型内容，围绕 AI wrapper、内容站、小游戏站、插件和社区项目是否值得做
- 4 篇左右真实复盘型内容，记录 0 下载、0 注册、0 流量、没人回复等结果后的调整
- 2 篇左右项目求反馈型内容，给新用户一个低成本参与入口
- 每篇重点帖配 1 到 3 条有信息量的回复
- 模拟点赞、收藏、关注和浏览记录只服务于真实内容的讨论氛围，不给低质量内容刷热度

### 2.3 先建立调性，再追求规模

早期最重要的指标不是帖子总数，而是用户点开几篇后是否觉得值得留下。

初始内容应避免：

- 测试帖
- 空泛大标题
- 纯新闻搬运
- 批量 SEO 长文
- 没有来源的热点整理
- 冒充个人经历的 AI 内容
- 没有具体问题的泛泛观点

## 3. 初始内容结构

### 3.1 第一层：门面帖

门面帖是用户判断社区质量的第一批内容，需要人工重点把关。

建议准备 8 到 10 篇，覆盖社区核心方向：

- Cursor、Codex、Claude Code 在真实项目里怎么分工
- 独立开发者现在还适合做 AI wrapper 吗
- 内容站冷启动先写内容，还是先找真实需求
- 小社区如何避免被 AI 水文淹没
- Open VSX 插件发布后没有下载量，应该先检查什么
- 一个人做产品时，哪些流程值得自动化
- SEO 流量下降后，小内容站还能靠什么增长
- 2C2G VPS 能不能撑起早期社区网站

门面帖标准：

- 标题具体
- 正文有场景、取舍、例子或判断
- 不追求大而全
- 结尾只问一个清晰问题
- 至少配一条能继续讨论的回复

### 3.2 第二层：讨论种子帖

讨论种子帖的目标不是写成文章，而是让用户容易插话。

建议准备 10 到 15 篇，例如：

- 你现在获取产品反馈最有效的渠道是什么
- Product Hunt 对小工具还有效吗
- 你会为 AI 工具付费，通常是因为什么
- 独立开发冷启动时，先做工具还是先写内容
- 用 Agent 辅助运营社区，哪些场景必须跳过
- 你现在最依赖的自动化脚本是什么
- 内容站文章写到多少篇后，才值得看 SEO 数据
- 小产品上线后，第一批用户通常从哪里来

讨论帖标准：

- 一个帖子只讨论一个问题
- 正文控制在 300 到 800 字
- 不要写成结论压死讨论
- 可以有站长观点，但要留出反驳空间

### 3.3 第三层：实用型长尾内容

这类内容承担搜索、收藏和后续引用价值。

建议准备 5 到 8 篇，例如：

- 国内服务器拉 Docker 镜像不稳定，可以怎么绕
- GitHub Actions 适合自动化哪些独立开发流程
- 内容站接 AdSense 前最好准备哪些页面
- 用 AI 总结外部文章时，怎样避免变成搬运
- 小社区上线前需要哪些基础规则
- 项目展示帖怎样写，别人更愿意给反馈

长尾内容标准：

- 有清晰步骤或判断标准
- 能被搜索到，也能被站内反复引用
- 必须标明外部来源
- 不复制外部正文和代码大段内容

## 4. 分类配比

首版不要平均填满所有分类。优先集中在 4 到 5 个主分类，让用户看到明确方向。

建议比例：

- AI 工具：30%
- 独立开发：30%
- SEO 与流量：20%
- 效率工具：15%
- 项目展示：5%

暂时不必强行填充：

- 闲聊
- 小游戏与内容站
- 测试类分类

分类可以空一点，但不能用低质量内容硬填。

## 5. 单篇内容质量门槛

每篇初始帖发布前至少满足以下条件：

- 标题具体，不使用泛泛大词
- 正文有一个明确场景
- 至少包含一个取舍、例子、流程或判断
- 没有伪造个人经历
- 外部信息保留来源链接
- 不复制外部文章主体内容
- 分类准确
- 标签 1 到 4 个
- 结尾有一个清楚的问题

建议质量分：

- 低于 70：不发布
- 70 到 84：可作为普通待审核内容
- 85 以上：可作为重点门面帖候选

## 6. 初始回复策略

零回复页面会让社区显得冷。上线前应给重点帖补少量高质量回复。

回复可以包括：

- 补充一个实际场景
- 提供另一个角度
- 提出一个可验证的小实验
- 追问作者一个具体问题
- 把抽象问题拆成更容易回答的选项

回复不应该只是：

- 支持
- 学习了
- 写得很好
- AI 式泛泛总结

建议：

- 每篇门面帖配 1 到 3 条回复
- 每篇讨论帖至少保留开放问题
- 如果回复不能增加信息量，宁可不回复

## 7. Agent 使用方式

Agent 可以帮助准备内容，但不应该完全自动铺量。

适合 Agent 做的事：

- 整理每日热点
- 总结外部文章并保留来源
- 生成讨论帖草稿
- 给无回复帖补充有信息量的回复
- 检查分类和标签
- 搜索站内近重复内容
- 记录 content run
- 提交具体质量风险建议

不适合 Agent 做的事：

- 冒充真实用户经历
- 批量发布低质量 SEO 文
- 改写外部文章当原创
- 没有查重就发帖
- 为了活跃度制造无意义回复

Agent 生成内容默认进入待审核状态。上线早期由站长筛选、修改和发布。

## 8. 上线前内容清理

正式上线前应清理或隐藏以下内容：

- 标题为“测试”的帖子
- 明显占位内容
- 无意义回复
- 低质量 AI 模板文
- 和社区定位无关的内容
- 过时或错误的操作说明

如果需要保留测试数据，应放在后台、草稿或开发环境，不要出现在公开首页。

## 9. 上线前内容清单

当前脚本已内置 51 篇种子话题，覆盖 6 个正式分类。

注意：这批内容是在功能开发阶段生成的临时内容池，只能用于验证分类、标签、用户、互动和后台统计链路。正式上线前应按最新需求研究重写为约 25 篇高信号内容，避免首页呈现批量 AI 水文感。

执行方式：

```bash
# 先确保数据库表结构、管理员和机器人账号已初始化
npm run db:init

# 正式上线前清空测试内容，重建分类、标签，并写入首批正式话题
npm run launch:reset

# 如果只想补写或刷新模拟用户
npm run launch:seed-users

# 如果只想补写或刷新首批话题，不清空其他内容
npm run launch:seed-content

# 如果只想补写或刷新互动数据
npm run launch:seed-interactions
```

当前内容池包含：

- 模拟用户：独立开发、AI 工作流、SEO、运维、写作和项目展示等画像
- AI 工具：Cursor、Codex、Claude Code、DeepSeek、Agent、AI 写作和模型接入
- 独立开发：MVP、第一批用户、定价、失败复盘、社区冷启动和域名备案
- SEO 与流量：内容站选题、Google 收录、AdSense、程序化 SEO 和访问统计
- 效率工具：GitHub Actions、Docker、PostgreSQL 搜索、知识库首页和番茄任务
- 项目展示：Leap Home、小游戏站实验、whyisee 后续功能反馈
- 互动数据：话题点赞、回复点赞、收藏、关注、浏览日志和话题浏览量

互动数据不包含：

- 举报
- 屏蔽
- 未读通知
- 邮件发送记录
- 封禁或审核异常

最低可上线标准：

- 首页首屏没有测试内容
- 最新 10 篇帖子质量稳定
- 每个核心分类至少 3 篇可读内容
- 至少 5 篇帖子有真实讨论感
- 站内规则、社区定位和新人入口清楚

## 10. 上线后两周节奏

正式上线后前两周不追求爆发，重点观察内容方向。

建议节奏：

- 每天 1 篇热点或外部文章整理
- 每周 2 到 3 篇经验型主帖
- 每周 1 篇项目展示或求反馈帖
- 每天检查无人回复的新帖
- 每周整理 1 次本周值得看的内容

重点观察：

- 哪类标题被点击
- 哪类问题有人回复
- 哪类内容被收藏或再次访问
- 哪些分类长期没人互动
- AI 生成内容是否影响社区质感

如果某个方向连续多篇没有互动，应减少该方向产出，把精力转到更具体的问题和案例。
