# 外部热榜扫描 Agent 设计

## 1. 目标

外部热榜扫描 Agent 用于从其他网站获取公开趋势信号，帮助社区发现值得讨论的选题。

第一版先实现知乎热榜扫描：

- 定时读取知乎热榜公开源。
- 标准化为站内 `external_hot_items`。
- 按来源和条目 ID 去重。
- 保留排名、热度、摘要、来源链接、首次发现和最近发现时间。
- 只入库，不自动发布帖子。

## 2. 为什么不直接自动发帖

热榜内容不等于社区内容。直接把外站热榜改写成帖子，容易变成低价值搬运，也会破坏社区调性。

正确流程应该是：

```text
外部热榜
  -> 入库去重
  -> 选题判断
  -> 资料核查
  -> 生成选题卡或草稿
  -> 人工审核
  -> 发布
```

第一版只完成前两步的基础设施：扫描和入库。

## 3. 当前实现

新增后台任务：

- `task_key`: `zhihu_hot_scan`
- `task_type`: `external_hot_scan`
- 默认状态：`paused`
- 默认间隔：`3600` 秒
- 默认来源：`https://rsshub.app/zhihu/hot`

新增数据表：

- `external_hot_items`

关键字段：

- `source`: 来源，例如 `zhihu_hot`
- `source_item_id`: 来源条目稳定 ID
- `title`: 标题
- `url`: 原始链接
- `summary`: 摘要
- `rank`: 热榜排名
- `heat_text`: 热度文本
- `first_seen_at`: 首次发现时间
- `last_seen_at`: 最近发现时间
- `seen_count`: 被扫描到的次数

## 4. 配置方式

进入后台：

```text
/admin/bot-jobs
```

找到“知乎热榜扫描”任务后可以配置：

- 状态：启用 / 暂停
- 间隔秒数
- 来源 URL
- 最大条数
- 超时毫秒
- User-Agent

也可以设置环境变量覆盖默认来源：

```bash
ZHIHU_HOT_SOURCE_URL=https://your-rsshub.example.com/zhihu/hot
```

建议生产环境优先使用自己部署的 RSSHub 或稳定代理源，不依赖公共服务。

## 5. 边界

当前版本不做：

- 不登录知乎。
- 不绕过验证码或访问控制。
- 不抓取需要认证的接口。
- 不自动发布话题。
- 不自动改写外站内容。

如果默认 RSS 源不可用，任务会失败并在“最近运行”里记录错误。

## 6. 下一步

建议后续增加 3 个能力：

1. 选题卡生成：把热榜条目转成“为什么值得讨论、适合哪个分类、可写什么角度”。
2. 站内查重：生成草稿前先搜索站内是否已有类似主题。
3. 一键转草稿：管理员从外部热榜列表中选择条目，生成待审核草稿。
