# whyisee.xyz 积分与勋章体系设计

## 1. 文档目标

本文档用于设计 `whyisee.xyz` 的用户激励体系，包括贡献值、信任等级和勋章成就。

目标不是制造刷分游戏，而是鼓励用户做出对社区长期有价值的行为：

- 发布有信息量的帖子
- 给项目提供具体反馈
- 分享真实复盘和实验记录
- 补充资料、案例和可验证信息
- 参与标签整理、内容治理和新人引导

## 2. 设计原则

### 2.1 三套体系各司其职

积分、等级和勋章不要混成一个简单排行榜。

- 贡献值：记录用户对社区的实际贡献
- 信任等级：决定用户可使用的社区权限
- 勋章成就：表达用户身份、擅长方向和里程碑

贡献值负责鼓励行为，信任等级负责建立秩序，勋章负责让用户被看见。

### 2.2 奖励质量，不奖励灌水

不要奖励单纯数量。早期社区最危险的激励是让用户觉得“发得越多越好”。

应优先奖励：

- 具体经验
- 真实数据
- 有判断的观点
- 对别人项目有帮助的反馈
- 能引发后续讨论的问题

应避免奖励：

- 无意义回复
- 纯表态内容
- 批量 AI 水文
- 重复搬运外部内容
- 互赞刷分
- 签到型低价值活跃

### 2.3 早期保持轻量

首版只需要能表达社区方向，不需要一次性做复杂系统。

首版建议：

- 一个公开贡献值
- 一个公开信任等级
- 20 个以内勋章
- 不做积分商城
- 不做复杂排行榜
- 管理员可以手动授予质量类勋章

## 3. 贡献值体系

贡献值用于衡量用户对社区的正向贡献。建议命名为“贡献值”，不要命名为金币、积分币或收益点数。

### 3.1 加分规则

| 行为 | 建议分值 | 说明 |
| --- | ---: | --- |
| 通过审核发布主题 | +10 | 普通主题通过审核后计分 |
| 发布项目展示帖 | +15 | 鼓励用户展示真实项目 |
| 发布复盘或实验记录 | +20 | 鼓励高密度实践内容 |
| 主题被设为精选 | +30 | 由管理员或高信任用户确认 |
| 发布有价值回复 | +5 | 需要达到基本质量门槛 |
| 回复被楼主标记为有帮助 | +15 | 鼓励解决问题和具体反馈 |
| 给项目帖提供具体反馈 | +10 | 反馈应包含问题、建议或验证路径 |
| 整理外部文章并保留来源 | +8 | 必须有原创摘要和讨论问题 |
| 完善个人资料 | +5 | 头像、简介、链接等基础资料 |
| 有效举报低质或违规内容 | +5 | 由管理员确认有效 |
| 协助整理标签或分类 | +5 | 适合 L2 以上用户 |

### 3.2 扣分规则

| 行为 | 建议扣分 | 说明 |
| --- | ---: | --- |
| 帖子被退回修改 | -5 | 内容方向可修正时使用 |
| 帖子被删除 | -10 到 -30 | 根据问题严重程度处理 |
| 重复发布近似内容 | -10 | 包括重复搬运和重复提问 |
| 明显 AI 水文 | -10 到 -20 | 无具体信息、无来源、无讨论价值 |
| 违规推广或垃圾内容 | -30 | 可叠加发帖限制 |
| 恶意互赞或刷回复 | -30 | 可叠加人工审核 |

扣分不是为了惩罚普通错误，而是给社区质量边界一个明确反馈。

## 4. 防刷规则

贡献值必须配合限制，否则会把用户引向低质量活跃。

建议规则：

- 普通回复每日最多计 30 分
- 同一话题内连续回复不重复计高分
- 同一用户之间的频繁互赞不计入高价值行为
- 点赞本身不直接产生大量贡献值
- 被删除内容撤销对应贡献值
- AI Agent 账号不参与普通用户排行榜
- 管理员和机器人账号应有单独标识
- 精选、样板帖、长期有用等质量奖励需要人工确认

不建议首版支持：

- 签到连续奖励
- 积分商城
- 积分兑换现金或权益
- 按发帖数量排行
- 按回复数量排行
- 热门帖作者无限吃分

## 5. 信任等级体系

信任等级用于逐步开放权限，不只是展示用户资历。

### 5.1 等级设计

| 等级 | 名称 | 建议条件 | 权益 |
| --- | --- | --- | --- |
| L0 | 观察者 | 新用户 | 浏览、少量回复、发帖需审核 |
| L1 | 记录者 | 贡献值 50；至少 1 个通过审核主题或 3 条有效回复 | 正常回复、收藏、基础发帖 |
| L2 | 实验者 | 贡献值 150；至少 3 篇通过审核内容 | 发布项目展示、使用更多标签 |
| L3 | 建造者 | 贡献值 500；多次精选或高质量反馈 | 推荐精选、协助整理标签 |
| L4 | 共建者 | 管理员邀请 | 参与治理、审核建议、资源共建 |
| L5 | 管理员 | 站点管理角色 | 站务、审核、分类维护、规则执行 |

### 5.2 升级条件

升级不应只看贡献值，还应结合质量和时间。

建议综合条件：

- 贡献值达到门槛
- 账号存在时间达到最低要求
- 最近没有严重违规
- 有一定数量的有效回复或主题
- 高等级需要管理员确认

### 5.3 权限开放顺序

早期建议先开放低风险能力：

- 收藏
- 正常回复
- 发布普通主题
- 发布项目展示
- 推荐精选
- 标签整理
- 审核建议

不建议早期开放：

- 删除他人内容
- 修改他人主题
- 直接发布免审核外链帖
- 大规模邀请权限

## 6. 勋章体系

勋章用于表达用户身份和贡献方向。勋章不一定都要自动发放，早期可以保留一部分人工授予。

### 6.1 起步勋章

| 勋章 | 条件 | 目的 |
| --- | --- | --- |
| 第一篇帖子 | 首次发布通过审核的主题 | 鼓励发出第一篇内容 |
| 第一次认真回复 | 首次有效回复通过质量门槛 | 鼓励参与讨论 |
| 完善资料 | 头像、简介、链接完整 | 鼓励真实身份感 |
| 首次项目展示 | 发布第一篇项目展示帖 | 鼓励展示真实项目 |

### 6.2 内容勋章

| 勋章 | 条件 | 目的 |
| --- | --- | --- |
| 实战记录者 | 发布 3 篇实践类内容 | 鼓励具体经验 |
| 复盘者 | 发布 3 篇复盘帖 | 鼓励失败和过程复盘 |
| 工具流作者 | 发布 3 篇 AI 或效率工具工作流 | 强化社区方向 |
| SEO 实验员 | 发布 3 篇 SEO 或流量实验内容 | 鼓励可验证增长实验 |
| 内容整理者 | 整理 5 篇外部文章并保留来源 | 鼓励原创摘要和引用规范 |

### 6.3 互动勋章

| 勋章 | 条件 | 目的 |
| --- | --- | --- |
| 破冰者 | 给 5 篇零回复帖留下有效回复 | 减少冷帖 |
| 认真反馈 | 给项目帖提供 5 次具体反馈 | 鼓励互助 |
| 好问题提出者 | 多个主题产生有效讨论 | 鼓励提出好问题 |
| 讨论续航者 | 让老帖重新产生讨论 | 鼓励沉淀内容复活 |
| 新人引导者 | 帮助新人完成首次发帖或项目展示 | 降低参与门槛 |

### 6.4 质量勋章

| 勋章 | 条件 | 目的 |
| --- | --- | --- |
| 精选作者 | 帖子被设为精选 | 鼓励高质量主题 |
| 社区样板帖 | 管理员手动授予 | 建立内容标杆 |
| 被收藏的内容 | 单帖收藏达到门槛 | 表示长期价值 |
| 长期有用 | 发布 30 天后仍有访问、收藏或回复 | 鼓励长尾价值 |
| 质量守门员 | 多次有效举报低质或违规内容 | 鼓励治理参与 |

### 6.5 周期勋章

周期勋章适合早期社区制造节奏，但不要永久拉开差距。

建议：

- 本周建设者
- 本月优质反馈
- 本月项目观察员
- 本月内容整理者
- 本月复盘作者

周期勋章可以在用户主页保留历史记录，但榜单只展示当前周期。

## 7. 榜单设计

榜单应鼓励贡献，而不是鼓励刷量。

首版可以做：

- 本周建设者
- 本周优质反馈
- 本月精选内容
- 最近获得勋章

不建议做：

- 总积分排行榜
- 回复数量排行榜
- 发帖数量排行榜
- 点赞数量排行榜

如果必须展示贡献值排行，应默认按周期展示，并排除机器人账号、管理员账号和异常账号。

## 8. Agent 与机器人账号规则

AI Agent 可以帮助社区运营，但不应参与普通用户成长体系。

用户账号与 Agent 子身份的展示、归属和内容来源边界见：[用户账号与 Agent 身份边界设计](../app/docs/agent-identity-boundary-design.md)。

建议规则：

- Agent 账号有明确标识
- Agent 发帖默认进入待审核
- Agent 不参与普通贡献榜
- Agent 不获得普通用户勋章
- Agent 的内容生产记录进入 content run
- Agent 可以触发质量建议，但最终由管理员确认
- 用户人工确认、修改、整理 Agent 草稿后，可以给少量整理贡献，但不按 Agent 自动产出的数量计分。

Agent 可以获得单独的系统标记，例如：

- 内容助手
- 搜索助手
- 审核建议助手

这些标记不和真人用户勋章混用。

## 9. 首版落地范围

第一版建议只做最小可用激励系统：

- 用户贡献值字段
- 用户信任等级字段
- 勋章定义表
- 用户勋章记录表
- 管理员手动授予和撤销勋章
- 发帖、回复、精选、删除时记录贡献变化
- 用户主页展示贡献值、等级和勋章

首版暂不做：

- 积分商城
- 勋章复杂动画
- 自动化全量风控
- 用户之间赠送积分
- 全站总积分排行榜

## 10. 数据模型草案

### 10.1 用户贡献字段

```text
users
- reputation_score
- trust_level
- reputation_updated_at
```

### 10.2 贡献流水

```text
reputation_events
- id
- user_id
- action
- points
- target_type
- target_id
- reason
- created_by
- created_at
```

贡献值应从流水聚合或至少保留流水，方便排查刷分和撤销错误奖励。

## 11. 运营复盘指标

上线后需要观察激励体系是否带来好行为。

重点看：

- 高质量回复占比是否上升
- 项目展示帖是否获得更多具体反馈
- 低质量短回复是否增加
- 用户是否为了积分拆分回复
- 勋章是否能帮助识别擅长领域
- 新用户是否更愿意完成第一次发帖或回复

如果出现刷分倾向，应优先降低数量型奖励，提高人工确认型质量奖励。

## 12. 推荐实施顺序

建议分三步做：

### 阶段 1：手动激励

- 用户贡献值
- 管理员手动加减贡献值
- 管理员手动授予勋章
- 用户主页展示

### 阶段 2：半自动激励

- 发帖通过审核自动加分
- 回复被标记有帮助自动加分
- 精选内容自动加分
- 删除内容自动撤销贡献值

### 阶段 3：社区共建

- L3 以上用户推荐精选
- L3 以上用户协助标签整理
- 周期榜单
- 长期有用内容识别
- 治理贡献勋章

早期宁可少给分，也不要让用户觉得社区是在鼓励刷存在感。
