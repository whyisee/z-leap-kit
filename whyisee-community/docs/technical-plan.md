# whyisee.xyz 技术方案规划

> 当前方向已从“优先选择现成论坛系统”调整为“先自研轻量内容社区 MVP”。新的主开发文档见：[whyisee.xyz 自研社区开发设计文档](./indie-community-development-design.md)。本文保留为早期技术选型记录。

## 1. 目标

为 `whyisee.xyz` 搭建一个类似 linux.do 的社区网站。优先选择成熟论坛系统，快速建立发帖、回复、分类、标签、搜索、通知、审核和 SEO 基础能力。

首版技术目标：

- 尽快上线可用社区
- 支持长期 SEO 收录
- 管理成本可控
- 后续可扩展资源页、项目库、精选内容页
- 不自研论坛核心功能

## 2. 关键约束

你已经购买：

- 域名：`whyisee.xyz`
- 主机：具体环境待确认

需要确认主机能力：

- 是否支持 Docker
- 内存大小
- CPU 核心数
- 是否有 root 权限
- 是否能配置 Nginx / Caddy
- 是否支持 PostgreSQL / Redis
- 是否是普通虚拟主机、VPS，还是云服务器
- 是否需要备案

## 3. 技术路线候选

### 方案 A：Discourse

推荐程度：最高，如果主机支持 Docker 且资源足够。

优点：

- linux.do 同类形态，成熟度高
- 分类、标签、话题流、通知、私信、信任等级完善
- SEO 基础好
- 管理后台成熟
- 插件生态强
- 社区规则、审核、反垃圾能力较完整

缺点：

- 对主机资源有要求
- 官方推荐 Docker 部署
- 初次配置稍复杂
- 邮件配置必须认真做

建议资源：

- 2GB 内存起步，4GB 更稳
- 1-2 核 CPU
- Docker
- 可配置 SMTP

适合：

- VPS / 云服务器
- 长期认真做社区

### 方案 B：Flarum

推荐程度：中等，如果主机是 PHP 虚拟主机。

优点：

- 比 Discourse 轻量
- PHP + MySQL 环境常见
- 界面简洁
- 插件不少
- 部署门槛相对低

缺点：

- 功能成熟度和治理能力弱于 Discourse
- SEO、权限、反垃圾、通知等需要插件补齐
- 长期社区治理能力不如 Discourse

适合：

- 普通虚拟主机
- 资源有限
- 想先快速跑起来

### 方案 C：NodeBB

推荐程度：中等偏低。

优点：

- Node.js 技术栈
- 实时体验好
- 可定制性强

缺点：

- 运维复杂度不一定低
- 生态和中文资料不如 Discourse / Flarum 直观
- 对“类似 linux.do”目标不是最直接

适合：

- 明确想用 Node 技术栈
- 愿意自己维护

### 方案 D：自研

推荐程度：不建议首版。

原因：

- 社区最难的是运营，不是写论坛代码
- 发帖、回复、通知、权限、审核、搜索、SEO、反垃圾都要长期维护
- 首版会被技术细节拖死

自研只适合作为后续：

- 项目库
- 精选页
- 资源目录
- 首页聚合
- 专题页

## 4. 推荐选择

优先级：

1. 如果主机是 VPS 且支持 Docker：用 Discourse
2. 如果主机是普通 PHP 虚拟主机：用 Flarum
3. 如果主机资源很弱：先做静态入口页 + 等换主机

当前建议：

> 优先尝试 Discourse，因为目标就是做 linux.do 类社区。不要一开始自研论坛。

## 5. Discourse 部署规划

具体执行步骤见：[Discourse 部署指南](./discourse-deployment-guide.md)。

### 5.1 基础架构

```text
whyisee.xyz
  -> DNS A/AAAA
  -> VPS
  -> Docker
  -> Discourse
  -> PostgreSQL + Redis
  -> SMTP 邮件服务
```

### 5.2 必备配置

- 域名解析到服务器
- HTTPS
- SMTP 发信
- 管理员账号
- 站点名称：Why I See
- 默认语言：中文
- 允许英文内容
- 分类和标签
- 社区规则页
- 隐私政策
- 服务条款

### 5.3 邮件

社区必须能发邮件，否则注册、找回密码、通知都会出问题。

可选 SMTP：

- Resend
- Mailgun
- SendGrid
- 腾讯企业邮箱
- 阿里云邮件推送

需要配置：

- SPF
- DKIM
- DMARC

## 6. SEO 规划

Discourse 自带较好的 SEO 能力，但要注意内容质量。

### 6.1 页面策略

重点让这些内容被收录：

- 高质量项目展示
- 上线复盘
- SEO 实验复盘
- AI 工具教程
- 插件/网站推广经验
- 小游戏站案例

### 6.2 标题规则

避免：

```text
求助
问一下
这个咋办
```

鼓励：

```text
Open VSX 插件发布后 7 天 0 下载，我做了哪些排查
用 GitHub Pages 做插件官网 SEO 的基础配置
小游戏站接 AdSense 前需要避开的 5 个坑
```

### 6.3 内容规则

要求高质量帖包含：

- 背景
- 目标
- 操作过程
- 截图/数据/链接
- 结论
- 下一步

## 7. 反垃圾和风控

首版必须配置：

- 新用户发帖频率限制
- 新用户链接数量限制
- 新用户图片上传限制
- 审核队列
- 敏感词和广告词观察
- 邀请制可选
- 邮箱验证

如果用 Discourse：

- 开启信任等级限制
- 配置 Akismet 或类似反垃圾能力
- 新用户前几帖进入审核

## 8. 主题和视觉

风格建议：

- 深色主题优先
- 简洁、论坛感、信息密度高
- 不做营销站风格
- Logo 可以先用文字：Why I See
- 强调“项目、复盘、实验”的实战感

首页话术：

```text
Why I See
看见想法，验证想法，做出东西。
```

## 9. MVP 上线步骤

### Step 1：确认主机环境

需要你提供：

```text
主机类型：
系统：
内存：
CPU：
是否有 root：
是否支持 Docker：
是否有控制面板：
```

### Step 2：选择论坛系统

- Docker VPS：Discourse
- PHP 虚拟主机：Flarum
- 资源不足：先换主机或做静态入口

### Step 3：配置域名

- `whyisee.xyz` A 记录指向服务器 IP
- 可选 `www.whyisee.xyz` CNAME 到主域
- 开启 HTTPS

### Step 4：安装社区系统

按选型执行安装。

### Step 5：初始化站点

- 创建管理员
- 设置站点名称
- 设置分类
- 设置标签
- 设置注册方式
- 设置邮件
- 设置规则页

### Step 6：发布种子内容

至少 8-10 个帖子再公开邀请用户。

### Step 7：开放小范围用户

先邀请真实用户测试：

- 发帖体验
- 邮件是否正常
- 搜索是否正常
- 中文标题是否正常
- 移动端是否能用
- 管理后台是否顺手
