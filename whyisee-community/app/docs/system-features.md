# whyisee-community 系统功能总览

更新时间：2026-06-04

本文档整理当前 whyisee-community 已具备的系统功能，便于后续继续开发、排期和查缺口。

## 系统定位

whyisee-community 是一个面向独立开发、AI 工具、效率工具、SEO 与内容站的小型社区系统。当前形态是自研 MVP，核心目标是先跑通内容沉淀、话题讨论、AI 辅助、用户互动和后台运营。

技术形态：

- Astro 服务端渲染应用。
- PostgreSQL 数据库。
- Markdown 内容写作与渲染。
- 自托管图片上传。
- 后台可配置 AI 模型，支持 OpenAI 兼容接口、DeepSeek、Anthropic、Google Gemini 和自定义服务商。

## 前台信息架构

### 首页与列表

- 首页 `/` 展示最新话题、分类、标签和精选话题。
- 最新页 `/latest` 展示话题流。
- 分类页 `/c/:slug` 展示某个分类下的话题。
- 标签页 `/tag/:slug` 展示某个标签下的话题，展示风格已对齐分类页。
- 项目页 `/projects` 展示项目类内容。
- 关于页 `/about`、社区规则页 `/guidelines` 提供基础说明。
- RSS `/rss.xml` 和站点地图 `/sitemap.xml` 已提供。

### 导航与布局

- 顶部 Header 包含侧边栏开关、品牌入口、新建话题、智能搜索、语言切换、通知和用户菜单。
- 左侧导航展示分类和标签，并支持折叠。
- 右侧边栏展示社区状态、精选话题等辅助信息。
- 话题 URL 已改为基于 ID 的 `/t/:id`，旧的 `/t/:id/:slug` 会重定向到 ID URL，避免中文标题进入 URL。

## 用户与账号

### 登录注册

- 支持登录、退出登录。
- 支持邀请制注册，需要邀请码。
- 用户创建后默认 active。
- 支持忘记密码、重置密码。
- 支持会话管理，修改密码后会撤销旧会话。

### 用户资料

- 用户主页 `/u/:username` 展示用户资料和话题。
- 个人资料页 `/settings/profile` 可维护显示名、简介、网站、GitHub、语言偏好。
- 账号页 `/settings/account` 可修改密码。

### 权限与状态

- 用户角色包括 `admin`、`moderator`、`member`、`new_user`。
- 用户状态包括 `active`、`pending`、`suspended`、`banned`。
- 管理员可在后台修改用户角色和账号状态。

## 内容发布

### 话题模型

话题包含：

- 标题、摘要、正文 Markdown。
- 分类。
- 多标签。
- 类型：讨论、问题、文章、项目、资源、公告。
- 状态：草稿、待审核、已发布、隐藏、删除。
- 置顶、精选。
- 浏览数、回复数、发布时间、最后活动时间。

### 新建话题

- 登录用户可从 Header 的写作图标进入 `/new-topic`。
- 新建话题页精简了顶部说明，重点保留写作表单。
- 从分类页或标签页进入新建话题时，会自动带入当前分类或标签。
- 普通用户提交话题后进入 `pending` 待审核状态。
- 管理员后台可直接创建草稿或发布内容。

### 编辑话题

- 管理员可在后台编辑话题全部字段。
- 普通用户可编辑自己的话题。
- 编辑保存后会重新渲染 Markdown、同步标签和提及关系。

### Markdown 与图片

- 正文和回复均支持 Markdown。
- 支持图片上传，接口为 `/api/uploads/images`。
- 上传文件通过 `/uploads/...` 访问。
- 支持 JPG、PNG、WebP、GIF，默认最大 5MB，可通过 `WHYISEE_UPLOAD_MAX_BYTES` 配置。
- Markdown 中连续图片会自动整理成图片网格，帖子内容图片不再单张占满整行。
- 有效的 `@username` 会渲染成用户主页链接。

## 话题详情与互动

### 话题详情页

- 详情页 `/t/:id` 展示分类、发布时间、浏览数、标题、摘要、标签、正文。
- 话题右侧展示话题信息和相关话题。
- 提供点赞、收藏、关注、举报入口。
- 登录用户可参与评论和回复。

### 评论系统

- 评论区展示评论总数。
- 支持最热和最新排序。
- 回复输入框位于评论区顶部。
- 评论很多并向下滚动时，回复控件支持固定在底部，便于持续回复。
- 输入框默认只展示文本框；聚焦后展示图片、@、表情、发布等工具按钮。
- 失焦后在内容为空时收起工具按钮。
- 支持上传图片、@ 用户或机器人、插入表情。
- 评论下方展示时间、点赞数、回复按钮。
- 鼠标移到评论上时显示更多操作入口，可复制评论链接、加入黑名单、举报。

### 评论嵌套

- 回复评论会折叠在父评论下展示。
- 默认展示每条评论下热度最高的一条子回复。
- 更多子回复通过“查看回复”展开。
- 子回复同样支持点赞、回复、举报等基础操作。

### 互动能力

- 支持话题点赞和评论点赞。
- 支持话题收藏。
- 支持关注话题，话题有新回复时通知关注者。
- 支持举报话题或评论。
- 支持拉黑用户；被拉黑用户的回复会在当前用户视角中过滤。

## @ 用户与 @ 机器人

### @ 用户

- 话题和回复中支持输入 `@username`。
- 编辑内容时会同步更新提及关系。
- 被提及用户会收到站内通知。
- 解析会跳过代码块、行内代码、链接等不应解析的位置。

### @ 机器人

- 机器人作为特殊用户存在，`users.is_bot = true`。
- 当前支持默认机器人方向：`@ai`、`@seo`、`@writer`、`@mod`。
- 用户提及机器人后会创建 `bot_jobs` 异步任务。
- 机器人任务会读取话题上下文和近期回复，调用默认 AI 模型生成机器人回复。
- 管理员可以在 `/admin/bot-jobs` 查看任务状态，并手动处理下一条队列任务。
- bot worker 脚本 `scripts/bot-worker.ts` 可用于持续处理队列。

## AI 功能

### AI 模型配置

- 管理后台 `/admin/ai` 可新增和编辑 AI 模型配置。
- 支持服务商：DeepSeek、OpenAI、Anthropic、Google、自定义。
- 可配置模型名、Base URL、API Key、温度、最大 Token、启用状态、默认模型。
- 系统 AI 功能统一使用启用中的默认模型。
- AI 请求有超时和错误分类，前端可展示未配置、缺少 API Key、超时等状态。

### 话题页 AI

话题详情页右上角提供 AI 工具菜单：

- 总结话题。
- 生成回复草稿。
- 双语翻译。
- 提炼行动项。
- 查找相关话题和标签方向。
- 内容质量检查。

AI 输出在话题页内展示，支持复制和关闭。

### 写话题 AI

新建和编辑话题表单内提供 AI 写作工具：

- 生成大纲。
- 续写正文。
- 生成标题。
- 润色内容。
- 去 AI 味。
- 生成摘要。
- 推荐分类和标签。

AI 写作结果可应用到标题、摘要、正文、分类和标签。

### AI 搜索

Header 搜索框已支持智能搜索。

普通搜索：

- 用户直接输入关键词或搜索指令。
- Header 浮层直接展示搜索结果。

指令搜索：

- 支持 `type:`、`in:`、`category:`、`tag:`、`#tag`、`author:`、`mention:`、`@user`、`after:`、`before:`、`within:`、`year:`。
- 支持 `has:image`、`has:link`、`has:reply`、`no:reply`、`is:pinned`、`is:featured`。
- 支持 `sort:relevance`、`sort:latest`、`sort:hot`、`sort:views`、`sort:replies`。

AI 搜索：

- 用户输入自然语言，例如“有图片的帖子”。
- 点击 Header 浮层内的“AI 搜索”。
- 系统先调用 `/api/search/translate`，把自然语言翻译成搜索指令，例如 `has:image`。
- 浮层单独展示 AI 解析后的指令，不覆盖原始输入。
- 系统再使用解析后的指令执行搜索并展示结果。

独立搜索页 `/search` 仍保留为完整搜索页面和兜底入口。

## 通知与邮件

### 站内通知

通知来源包括：

- 话题收到回复。
- 评论收到回复。
- 内容被点赞。
- 话题关注后有新回复。
- 被 `@username` 提及。
- 机器人回复。
- 管理员收到举报。

用户可在 `/notifications` 查看通知，并标记为已读。
Header 用户菜单展示未读通知数量。

### 邮件日志与发送

- 通知可按需触发邮件发送。
- 邮件发送会写入 `email_logs`。
- 邮件驱动支持 `console`、`smtp`、`disabled`。
- SMTP 支持账号密码认证和 TLS。

## 搜索系统

### 搜索范围

搜索结果覆盖：

- 话题。
- 回复。
- 分类。
- 标签。
- 用户。

### 搜索执行

- 普通关键词使用 PostgreSQL `to_tsvector('simple')` 和模糊匹配。
- 指令搜索会解析成 `SearchPlan`，再由后端用参数化 SQL 执行。
- AI 只负责把自然语言翻译成受控 JSON/指令，不直接生成 SQL。
- 搜索结果链接统一使用 `/t/:id`。

### 搜索 API

- `GET /api/search?q=...&mode=auto|directive|ai`
- `POST /api/search/translate`
- `GET /search?q=...`

## 管理后台

后台入口 `/admin`，仅管理员可访问。

### 话题管理

- 按状态、分类、标签、类型、关键词筛选话题。
- 创建、编辑话题。
- 发布、退回草稿、隐藏、删除。
- 置顶、取消置顶。
- 精选、取消精选。

### 分类管理

- 新增分类。
- 编辑名称、slug、描述、颜色、排序、公开状态。
- 显示分类下话题数量。

### 标签管理

- 新增标签。
- 编辑名称、slug、描述。
- 显示标签下话题数量。

### 用户管理

- 搜索用户。
- 查看用户邮箱、创建时间、最后登录时间。
- 修改角色和账号状态。

### 举报管理

- 查看举报列表。
- 按状态筛选。
- 更新举报状态：open、accepted、rejected、resolved。

### AI 配置

- 新增和编辑模型配置。
- 管理默认模型。
- 启用、停用模型。
- 保存或清空 API Key。

### 机器人任务

- 查看 bot job 列表。
- 按状态筛选 queued、running、succeeded、failed。
- 手动处理下一条 queued 任务。

### 运营统计

- 总览用户数、话题数、回复数、待处理举报、待审核话题。
- 今日访问量、独立访客、新增用户、新增话题、新增回复。
- 7 天热门页面。
- 举报状态统计。

## 当前边界与注意事项

- 社区仍处于自研 MVP 阶段，内容审核和运营流程以后台手动处理为主。
- 普通用户发帖默认进入待审核，不是直接发布。
- AI 能力依赖后台配置默认模型和 API Key。
- AI 搜索现在主要集成在 Header 搜索浮层，`/search` 是完整页面兜底。
- 机器人任务可以自动触发，也可以由管理员手动处理；生产环境建议运行 bot worker。
- 图片上传为本地文件存储，生产部署需要规划持久化目录和备份。
- 邮件默认可使用 console 模式，生产环境需要配置 SMTP。
- 搜索是全文和规则指令搜索，还没有向量索引和语义召回。

## 后续演进路线

系统后续应该围绕“内容质量、讨论效率、AI 辅助、搜索发现、运营治理、基础设施”几条主线演进。优先级建议是先把内容和互动闭环跑稳，再放大 AI 与搜索能力，最后补齐增长、数据和规模化能力。

### 内容生产与审核

- 完善普通用户发帖审核流程：待审核列表、审核原因、通过/驳回通知、批量审核。
- 增加草稿箱和个人内容管理，让用户能管理自己的草稿、待审核、已发布、被隐藏内容。
- 支持话题版本历史，方便追踪修改记录和恢复误改内容。
- 增加内容质量评分：标题是否具体、正文是否有上下文、标签是否合理、是否疑似灌水。
- 支持编辑建议模式，管理员或 AI 可以给作者留下修改建议，而不是只有通过/隐藏。
- 增加内容模板：项目展示、问题求助、复盘、资源推荐、工具测评等。

### 评论与社区互动

- 完善评论回复体验：楼中楼展开状态记忆、跳转到指定评论、高亮被引用评论。
- 增加评论引用能力，支持回复时引用选中文本。
- 增加更多互动类型，例如感谢、有用、收藏评论。
- 增加用户关注能力：关注用户后可在个人动态或通知中看到新话题。
- 增加个人动态流：我的回复、我的收藏、我关注的话题、提到我的内容。
- 增加黑名单管理页，让用户可以查看和移除已拉黑用户。

### AI 能力增强

- 把 AI 搜索从“自然语言转指令”升级为“搜索、总结、追问”三段式体验。
- 支持搜索结果 AI 汇总：用户搜到多条话题后，AI 总结共同结论和差异。
- 增加话题内 AI 问答：基于当前话题和回复回答问题，不离开话题页。
- 增加 AI 标签治理：识别重复标签、建议合并标签、补全标签描述。
- 增加 AI 审核助手：识别广告、低质内容、辱骂、重复灌水、疑似机器人内容。
- 增加机器人人格和权限配置：每个机器人可配置能力范围、触发条件、每日调用额度。
- 增加 AI 调用日志与成本统计：按用户、功能、模型统计调用次数、Token、失败率。

### 搜索与内容发现

- 为搜索结果增加分组展示：话题、回复、用户、标签、分类分区展示。
- 增加搜索指令帮助浮层，用户输入 `?` 或点击提示时展示可用指令。
- 增加搜索历史和常用搜索收藏。
- 增加相关搜索建议：根据关键词推荐标签、分类、相似查询。
- 引入向量索引或混合检索，用于语义搜索和相似话题推荐。
- 增加“相似问题已存在”提醒，在用户发帖时自动提示相关话题。
- 优化站内推荐：按分类、标签、阅读历史、互动热度生成推荐列表。

### 用户成长与社区秩序

- 增加用户等级、徽章和信誉分，但要避免刷积分导向。
- 基于信誉分控制能力：新用户发帖审核更严格，高信誉用户可更快通过。
- 增加邀请体系管理：邀请码生成、使用记录、邀请关系、邀请额度。
- 增加社区规则执行记录：警告、禁言、封禁、申诉。
- 支持用户举报后的处理反馈，让举报人知道结果。
- 增加敏感操作审计：管理员修改用户、话题、AI 配置时写入 audit logs。

### 管理后台与运营

- 管理后台增加待办首页：待审核话题、待处理举报、失败机器人任务、异常 AI 调用。
- 增加运营看板：新增内容、活跃用户、评论率、搜索关键词、热门标签趋势。
- 增加内容运营工具：批量加标签、批量改分类、批量精选、批量隐藏。
- 增加邮件和通知模板管理，便于统一通知文案。
- 增加数据导出：用户、话题、评论、举报、访问统计。
- 增加后台权限细分：管理员、版主、运营、只读观察者。

### 基础设施与稳定性

- 将图片上传目录迁移到可持久化存储，并建立备份策略。
- 增加定时任务体系，用于机器人任务、邮件重试、统计汇总、清理过期会话。
- 增加队列系统，避免 AI、邮件、通知、上传处理阻塞用户请求。
- 增加日志与错误监控：接口错误、AI 错误、SMTP 错误、数据库慢查询。
- 增加速率限制：登录、注册、发帖、回复、AI 调用、上传、搜索。
- 增加缓存策略：分类、标签、精选话题、热门话题、搜索建议。
- 增加数据库备份恢复演练，确保线上数据可恢复。

### SEO 与增长

- 完善话题页 SEO：结构化数据、Open Graph、摘要优化、canonical。
- 增加标签页和分类页的 SEO 描述管理。
- 增加站内热门内容榜单：今日热门、本周热门、最受收藏、最多回复。
