# 倒计日/时组件设计文档

## 目标

倒计日/时组件用于把重要日期、截止时间和阶段性节点放到 Leap Home 首页，让用户打开工作台时能立刻看到“还有多久”和“哪个节点最接近”。它不是日历的重复视图，而是对少量高优先级时间节点的持续提醒。

## 产品定位

倒计日/时组件解决三类问题：

- 重要日期：发布、考试、旅行、纪念日、合同到期。
- 关键截止：需求评审、上线窗口、交付节点、报名截止。
- 过程提醒：距离某个时间点还有多少天、多少小时，帮助用户提前安排任务。

组件应该保持克制，只展示真正值得被盯住的节点。普通日程仍然放在周历/月历里。

## MVP 范围

- 新增组件 `countdown`。
- 支持倒计日和倒计时两种模式：
  - 倒计日：只配置日期，展示剩余天数。
  - 倒计时：配置日期和时间，展示天/小时/分钟。
- 支持新增、编辑、删除倒计项。
- 支持标记完成。
- 支持过期状态展示。
- 默认按距离目标时间从近到远排序。
- 首页最多展示 3 到 5 个最近节点。
- 数据存储在 `.leap/components/countdown.json`。

## 暂不做

- 重复规则，例如每年生日、每周例会。
- 系统通知和声音提醒。
- 与外部日历同步。
- 拖拽排序。
- 多时区。
- AI 自动从文档中提取 deadline。

## 数据模型

```json
{
  "version": 1,
  "items": [
    {
      "id": "countdown-...",
      "title": "MVP 版本发布",
      "mode": "datetime",
      "targetDate": "2026-06-15",
      "targetTime": "18:00",
      "note": "发布前需要完成搜索、四象限和番茄时钟验证",
      "color": "blue",
      "done": false,
      "createdAt": "2026-05-31T10:00:00.000Z",
      "updatedAt": "2026-05-31T10:00:00.000Z"
    }
  ]
}
```

字段说明：

- `mode=date`：只按本地日期计算剩余天数。
- `mode=datetime`：按本地日期和时间计算剩余时间。
- `targetDate`：必填，格式 `YYYY-MM-DD`。
- `targetTime`：可选，格式 `HH:mm`。
- `color`：用于视觉区分，先提供少量内置颜色。
- `done`：已完成后默认不展示在主列表，可通过开关查看。

## 计算规则

以用户本地时间为准。

- `date` 模式：
  - 目标日期晚于今天：展示 `还有 N 天`。
  - 目标日期等于今天：展示 `今天`。
  - 目标日期早于今天：展示 `已过 N 天`。
- `datetime` 模式：
  - 大于 24 小时：展示 `N 天 H 小时`。
  - 小于 24 小时：展示 `H 小时 M 分钟`。
  - 小于 1 小时：展示 `M 分钟`。
  - 已过期：展示 `已过期` 和超时长度。

排序规则：

1. 未完成优先。
2. 未过期优先。
3. 距离当前时间最近优先。
4. 已过期按过期时间从近到远。

## 交互设计

组件主体：

- 顶部展示最近的核心倒计项，用更大的数字突出剩余时间。
- 下方展示 2 到 4 个次级倒计项。
- 每项显示标题、剩余时间、目标日期时间和颜色标识。
- 鼠标移入展示编辑、完成、删除按钮。

新增/编辑：

- 默认只显示一个 `+` 按钮。
- 点击后展开表单：标题、日期、时间、颜色、备注。
- 保存后收起。
- 编辑使用同一套表单。

状态：

- 正常：展示剩余时间。
- 今天/24 小时内：强调。
- 过期：弱化但可见。
- 已完成：默认隐藏。

## 与 Leap Home 的关系

- 倒计项是独立数据，不进入搜索索引。
- 后续可以从四象限任务的 `dueDate` 和月历事件中快速创建倒计项。
- 后续统计组件可以展示“最近节点”和“已过期倒计项”。
- 做什么推荐组件可以把临近倒计项作为推荐权重来源。
