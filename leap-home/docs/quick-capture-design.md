# 快速记录组件设计文档

## 目标

快速记录组件用于承接用户打开 Leap Home 时的临时输入：想法、待办、链接、代码片段。它不应该只是“打开一个输入框”，而应该成为个人知识库首页的低摩擦入口，把内容按合适去向沉淀到收集箱或任务系统。

## MVP 范围

- 页面内直接输入内容，不再只依赖 Cursor 命令弹窗。
- 支持类型：想法、待办、链接、代码片段。
- `Ctrl/Cmd + Enter` 快速保存。
- 想法、链接、代码片段写入默认收集箱。
- 待办写入四象限，默认进入“重要不紧急”。
- AI 按钮用于把输入作为待办交给 DeepSeek 判断四象限和截止日期。
- 最近记录展示保存结果、去向、时间和简短内容。
- 最近记录数据存储在工作空间 `.leap/components/quick-capture.json`。
- 保留打开收集箱入口。

## 暂不做

- 完整 AI 多去向路由：自动判断是收集箱、任务、日历事件或 Prompt。
- 附件、图片、剪贴板富文本捕获。
- 拖拽记录到组件。
- 记录编辑、撤销和批量整理。
- 独立收集箱看板。

## 数据设计

收集箱仍使用现有配置：

- 默认路径：`.leap/inbox.md`
- 可通过 Cursor 设置 `leapHome.inboxPath` 覆盖。

快速记录历史独立存储：

```json
{
  "version": 1,
  "items": [
    {
      "id": "capture-...",
      "text": "记录正文",
      "kind": "task",
      "target": "quadrant",
      "label": "重要不紧急",
      "dueDate": "2026-05-30",
      "reason": "AI 归类理由",
      "createdAt": "2026-05-30T00:00:00.000Z"
    }
  ]
}
```

## 交互

- 主体是一个多行输入框，默认适配短想法和代码片段。
- 类型选择和日期配置放在输入框下方，减少首屏噪音。
- 保存按钮按当前类型执行：
  - 想法、链接、代码片段：格式化后追加到收集箱。
  - 待办：写入四象限“重要不紧急”。
- AI 按钮把当前内容作为待办进行 AI 四象限归类。
- 最近记录只展示最近几条，帮助用户确认“刚才写到哪里了”。
