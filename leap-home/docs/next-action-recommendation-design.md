# 做什么推荐组件设计文档

## 目标

做什么推荐组件用于在用户打开 Leap Home 时给出“现在最应该做什么”的建议。它不是简单列待办，也不是 AI 随口建议，而是根据四象限、日历、倒计项、番茄时钟、快速记录和知识上下文，给出少量可行动推荐。

## 产品定位

用户真正需要的不是更多列表，而是减少选择成本。推荐组件应该回答：

- 现在最紧急的事是什么？
- 当前有精力时，做哪件事最能推进目标？
- 有没有马上过期、已经过期、被忽略太久的事项？
- 如果刚完成一个番茄，下一步应该继续、休息，还是切换任务？

组件必须可解释。每条推荐都要说明为什么推荐，以及下一步动作是什么。

## MVP 范围

- 新增组件 `nextAction`。
- 本地规则生成推荐，不依赖 AI。
- 支持手动 AI 行动教练：基于最近笔记、未完成任务、倒计节点、番茄记录和搜索历史生成行动建议。
- AI 可以重排本地候选，也可以生成新的小任务、笔记整理建议、鼓励和创新探索方向。
- 系统推荐和 AI 建议通过 tab 切换展示，避免混淆来源且节省组件高度。
- AI 建议支持输入问题，例如“我现在应该先做什么”“帮我拆一个不想开始的任务”。
- 默认按组件 `limit` 展示推荐，模板默认 4 条，用户可在编辑模式中调整。
- 推荐来源：
  - 四象限未完成事项。
  - 四象限事项的截止日期。
  - 倒计日/时中临近或过期节点。
  - 快速记录中的待整理内容。
  - 番茄时钟状态。
  - 最近搜索主题。
- 推荐类型：
  - `do-now`：现在做。
  - `plan`：需要安排。
  - `review`：需要整理/复盘。
  - `break`：需要休息。
- 支持一键动作：
  - 开始番茄。
  - 标记任务完成。
  - 跳转到相关组件或搜索命令。
  - 暂时忽略推荐。
- 记录推荐曝光、采纳、忽略和置顶事件，用于后续统计采纳率与准确率。
- 用户反馈存储在 `.leap/components/next-action.json`。

## 暂不做

- AI 自动规划一天。
- 自动修改任务日期。
- 与外部日历同步。
- 长期习惯建模。
- 跨工作区推荐。
- 推荐通知。

## 数据输入

推荐组件不维护任务本体，只读取其他组件数据。

当前已使用：

- 四象限：任务内容、象限、截止日期、完成状态、AI 归类理由。
- 倒计项：目标日期、过期状态、剩余时间。
- 快速记录：最近记录、待整理内容。
- 番茄时钟：当前状态、本轮专注时长、打断次数。
- 搜索历史：用户最近关注主题。

已预留但还没有进入推荐打分：

- 日历事件：今日、本周事件。
- 收藏：高价值文档上下文。
- 知识图谱：相关文档关系。

## 打分规则

MVP 使用本地规则打分，确保没有 AI 配置时也可用。

任务基础分：

- 重要且紧急：`+50`
- 重要不紧急：`+34`
- 不重要但紧急：`+24`
- 不重要不紧急：`+8`

时间权重：

- 已过期：`+34`
- 今天截止：`+30`
- 明天截止：`+22`
- 3 天内截止：`+16`
- 本周截止：`+10`
- 无日期但重要：`+8`

当前已实现的上下文权重：

- 与最近搜索主题匹配：`+8`
- 重要但没有日期：`+8`
- 临近倒计项独立生成候选，基础分 `54 + 时间权重`。
- 快速记录独立生成整理候选，基础分从 `42` 递减。

待接入权重：

- 任务和临近倒计项的语义关联：`+14`
- 出现在今日/月历详情中：`+16`
- 收藏文档或知识图谱中有相关文档：`+8`

当前已实现的番茄状态推荐：

- 专注番茄完成后，生成休息推荐。
- 休息完成后，生成回到待办的推荐。

待接入番茄权重：

- 当前没有番茄在跑，且推荐是 `do-now`：`+8`
- 本轮打断次数过多：推荐缩小任务粒度 `+10`

惩罚：

- 用户刚忽略：`-100`
- 已完成：排除。
- 距离截止超过 30 天：`-10`

待接入惩罚：

- 同类推荐过多：保留最高分，其他降权。

最终排序：

1. 置顶推荐。
2. 分数高优先。
3. 过期/今日优先。
4. 重要优先。
5. 标题稳定排序。

## 推荐生成流程

```text
read model data
  -> build candidates from tasks / countdown / quick capture / focus timer
  -> score candidates
  -> apply dismiss / pin feedback
  -> return top N by component limit
```

推荐在扩展端生成，webview 只展示结果和发送动作消息。

## 交互设计

组件默认使用两个 tab：

- 系统推荐：只展示本地规则计算出的推荐，强调稳定、可解释、不依赖 AI。
- AI 建议：展示 DeepSeek 根据当前上下文和用户问题生成的行动教练建议。

AI 建议区提供输入框：

- 输入问题后点击提问或回车，向 DeepSeek 发送当前任务、最近笔记、倒计节点、番茄记录和搜索历史。
- 不输入问题时，按钮按默认“现在应该做什么”逻辑生成建议。
- 上次问题、上下文总结和鼓励语会展示在 AI tab 顶部。

两个 tab 共用同一块内容面板，避免在小尺寸组件里上下堆叠占用空间。

主推荐：

- 标题：建议做什么。
- 原因：一句话解释。
- 动作：开始番茄、完成、查上下文。

次推荐：

- 紧凑展示标题、来源和分数原因。
- 直接展示可执行动作，点击后进入处理中状态，避免重复触发。

状态：

- 没有推荐：展示“当前没有高优先级事项”，并提供打开四象限或快速记录入口。
- 番茄刚完成：优先展示休息/复盘推荐。
- 推荐被采纳后会临时隐藏，避免反复展示同一条 AI 建议或同一条已处理事项。

## 推荐类型

### `do-now`

适合立即执行。通常来自重要且紧急、今日截止、过期任务。

### `plan`

适合安排日期或拆解。通常来自重要但无日期、临近倒计项。

### `review`

适合整理。当前通常来自快速记录，后续会接入搜索历史和孤立知识图谱节点。

### `break`

适合休息。通常来自番茄完成或打断过多。

## 与 Leap Home 的关系

- 四象限提供任务主体。
- 周历/月历提供事项展示和日期入口；推荐打分当前主要使用四象限事项截止日期。
- 倒计日/时提供关键节点压力。
- 番茄时钟提供执行入口。
- 知识图谱后续提供相关文档上下文。
- 搜索组件用于“查上下文”动作。

推荐组件应该成为首页的行动入口，而不是另一个列表组件。

## 统计指标

事件流保存在 `.leap/components/next-action.json` 的 `events` 中，最多保留最近 500 条。

- `impression`：推荐在当前 tab 中展示过，按“日期 + 推荐 key + 来源 + AI 批次”去重。
- `adopted`：用户点击了推荐动作，例如开始番茄、完成、查上下文、加入待办。
- `dismissed`：用户明确忽略推荐。
- `pinned` / `unpinned`：用户置顶或取消置顶。

派生指标：

- 采纳率：`adopted / impression`。
- 准确率：`adopted / (adopted + dismissed)`，只统计用户有明确反馈的推荐。
- AI 采纳数：AI 建议被采纳次数。
- 系统采纳数：本地规则推荐被采纳次数。
