# 做什么推荐组件设计文档

## 目标

做什么推荐组件用于在用户打开 Leap Home 时给出“现在最应该做什么”的建议。它不是简单列待办，也不是 AI 随口建议，而是根据四象限、日历、倒计项、番茄时钟、快速记录和知识上下文，给出少量可行动推荐。

## 产品定位

用户真正需要的不是更多列表，而是减少选择成本。推荐组件应该回答：

- 现在最紧急的事是什么？
- 当前有精力时，做哪件事最能推进目标？
- 有没有马上过期、已经过期、被忽略太久的事项？
- 如果刚完成一个番茄，下一步应该继续、休息，还是切换任务？

组件必须可解释。每条推荐都要说明为什么推荐，以及下一步动作是什么。

## MVP 范围

- 新增组件 `nextAction`。
- 本地规则生成推荐，不依赖 AI。
- 每次最多展示 3 条推荐。
- 推荐来源：
  - 四象限未完成事项。
  - 周历/月历中的今日和本周事项。
  - 倒计日/时中临近或过期节点。
  - 快速记录中的待整理内容。
  - 番茄时钟状态。
- 推荐类型：
  - `do-now`：现在做。
  - `plan`：需要安排。
  - `review`：需要整理/复盘。
  - `break`：需要休息。
- 支持一键动作：
  - 开始番茄。
  - 标记任务完成。
  - 跳转到相关组件或搜索命令。
  - 暂时忽略推荐。
- 用户反馈存储在 `.leap/components/next-action.json`。

## 暂不做

- AI 自动规划一天。
- 自动修改任务日期。
- 与外部日历同步。
- 长期习惯建模。
- 跨工作区推荐。
- 推荐通知。

## 数据输入

推荐组件不维护任务本体，只读取其他组件数据：

- 四象限：任务内容、象限、截止日期、完成状态、AI 归类理由。
- 日历：今日、本周事件。
- 倒计项：目标日期、过期状态、剩余时间。
- 快速记录：最近记录、待整理内容。
- 番茄时钟：当前状态、本轮专注时长、打断次数。
- 搜索历史：用户最近关注主题。
- 收藏：高价值文档上下文。

用户反馈单独存储：

```json
{
  "version": 1,
  "dismissed": [
    {
      "key": "task:task-123",
      "until": "2026-05-31T18:00:00.000Z",
      "reason": "not-now"
    }
  ],
  "pins": [
    {
      "key": "task:task-456",
      "createdAt": "2026-05-31T10:00:00.000Z"
    }
  ]
}
```

## 推荐对象模型

```json
{
  "id": "recommendation-...",
  "key": "task:task-123",
  "type": "do-now",
  "title": "先完成搜索组件剩余 TODO",
  "reason": "重要且紧急，截止日期是今天，且已经被月历标记为今日事项。",
  "score": 92,
  "source": {
    "type": "task",
    "quadrantId": "importantUrgent",
    "taskId": "task-123"
  },
  "actions": [
    { "type": "startFocus", "label": "开始番茄" },
    { "type": "completeTask", "label": "完成" },
    { "type": "search", "label": "查上下文", "query": "搜索组件 TODO @docs" }
  ]
}
```

## 打分规则

MVP 使用本地规则打分，确保没有 AI 配置时也可用。

任务基础分：

- 重要且紧急：`+50`
- 重要不紧急：`+34`
- 不重要但紧急：`+24`
- 不重要不紧急：`+8`

时间权重：

- 已过期：`+34`
- 今天截止：`+30`
- 明天截止：`+22`
- 3 天内截止：`+16`
- 本周截止：`+10`
- 无日期但重要：`+8`

上下文权重：

- 出现在今日/月历详情中：`+16`
- 关联临近倒计项：`+14`
- 与最近搜索主题匹配：`+8`
- 收藏文档或知识图谱中有相关文档：`+8`
- 快速记录刚捕获但未整理：`+10`

番茄状态权重：

- 当前没有番茄在跑，且推荐是 `do-now`：`+8`
- 当前番茄已完成：推荐休息或复盘 `+24`
- 本轮打断次数过多：推荐缩小任务粒度 `+10`

惩罚：

- 用户刚忽略：`-100`
- 已完成：排除。
- 距离截止超过 30 天：`-10`
- 同类推荐过多：保留最高分，其他降权。

最终排序：

1. 置顶推荐。
2. 分数高优先。
3. 过期/今日优先。
4. 重要优先。
5. 创建时间早的任务优先。

## 推荐生成流程

```text
read model data
  -> build candidates from tasks / calendar / countdown / quick capture / focus timer
  -> score candidates
  -> apply dismiss / pin feedback
  -> merge similar recommendations
  -> return top 3
```

推荐在扩展端生成，webview 只展示结果和发送动作消息。

## 交互设计

组件默认展示 1 条主推荐和 2 条次推荐。

主推荐：

- 标题：建议做什么。
- 原因：一句话解释。
- 动作：开始番茄、完成、查上下文。

次推荐：

- 紧凑展示标题、来源和分数原因。
- 鼠标移入展示忽略按钮。

状态：

- 没有推荐：展示“当前没有高优先级事项”，并提供打开四象限或快速记录入口。
- 番茄刚完成：优先展示休息/复盘推荐。
- 所有推荐都被忽略：展示下一次恢复时间。

## 推荐类型

### `do-now`

适合立即执行。通常来自重要且紧急、今日截止、过期任务。

### `plan`

适合安排日期或拆解。通常来自重要但无日期、临近倒计项。

### `review`

适合整理。通常来自快速记录、搜索历史、孤立知识图谱节点。

### `break`

适合休息。通常来自番茄完成或打断过多。

## 与 Leap Home 的关系

- 四象限提供任务主体。
- 周历/月历提供时间上下文。
- 倒计日/时提供关键节点压力。
- 番茄时钟提供执行入口。
- 知识图谱提供相关文档上下文。
- 搜索组件用于“查上下文”动作。

推荐组件应该成为首页的行动入口，而不是另一个列表组件。
